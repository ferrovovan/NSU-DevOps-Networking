# Решение задач по «Введение в компьютерные сети»

## Задания

### 1. PING
Пингануть 10 доменов, вывести результаты в csv таблице.
В качестве результатов предоставить RTT и 3 других параметра на ваше усмотрение.

### 2. SSH + Wireshark
Реализовать доступ на любой сервер сервер
  (например по параллелькам) по SSH без ввода пароля,
  а также пуш в удаленный репозитория без ввода пароля.
Захватить сетевой трафик с помощью инструмента
   __tcpdump__ или __Wireshark__ во время подключения по SSH.
Найти этапы установления соединения (TCP handshake),
   шифрование данных и завершение сессии.

### 3. Parser
Написать python-скрипт - парсер, используя SELENIUM или PLAYWRITE.
- Сайт обязательно должен иметь пагинацию.
- И, желательно, авторизацию.
- Парсим что угодно, новостной сайт, интернет-магазин, - без разницы.
- Параметров сохраняет не менее 4ёх.
- Записывает всё в csv файл.
Далее он же будет завязан на следующем задании.

### 4. Fast API
Поднять API точку, перейдя по которой,
  или дернув через curl,
  передав параметры пагинации (например: `127.0.0.1/parse?url=https://site.com/page`),
  запустится ваш парсер, отработает, запишет результат в базу данных.
Отдельным методом должны извлекаться данные из ТАБЛИЦЫ БД и отдаваться в виде json.

- БД любая (например PostgresSQL), но __не SQLite__.


### 5. Docker
Развернуть отдельно 2 docker контейнера, (БЕЗ  docker-compose) и настроить сеть между ними.
- 1ый контейнер - ваше приложение (на порту отличном от 80)
- 2ой контейнер - база данных

p.s. Вырезать парсер; первый метод с url теперь
  просто записывает этот url в базу данных,
  а второй метод возвращает все url, которые есть в БД.

### 7. Geo-Redirect
Прикрутить заглушку по Российскому IP / prefix.
То есть, переходя с РФ ip должен сработать РЕДИРЕКТ
  на любую заглушку вроде "ВАМ СЮДА НЕЛЬЗЯ".
- Важно! Заглушка должна быть реализована на уровне сервера, а __не вашего приложения__.


### 8. Tunnel Vision (Remote Access)
Сделать так, чтобы ваше приложение было доступно на моём пк,
  то есть вы присылаете мне ip, и у меня открывается ваш проект.
Метод реализации этого ничем не ограничен.

- Это задание для опробывания VPN. Например, WireGuard.
- Но можно и через ssh-forwarding :)
- VPN - это __клиент-серверная__ архитектура;
- Прокси - __серверная__.

### 9. April Fools' Day:  IPv6 vs IPv4
Настроить IPv6 на виртуальных машинах или контейнерах,
  и сравнить пакеты IPv4 и IPv6,
результатом word файл, со сравнением и кратким резюме.
